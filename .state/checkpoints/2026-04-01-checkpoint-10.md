### 关键发现摘要

#### 1. API 服务 (services/api/)

**多 Provider 支持**:
- First-Party Anthropic (api.anthropic.com)
- AWS Bedrock
- GCP Vertex
- Azure Foundry

**重试策略** (withRetry.ts):
- 指数退避 + jitter (base 500ms, max 32s)
- 无人值守模式持久重试 (最长 5min 退避)
- 429/529 快速回退
- 3 次 529 错误后模型降级 (Opus → Sonnet)

**错误分类** (errors.ts):
- 25+ 特定错误分类
- 上下文错误消息与恢复提示
- 交互式 vs 非交互式不同处理

**速率限制** (claudeAiLimits.ts):
- Header-based 限制提取
- 早期预警系统
- 5 小时和 7 天窗口跟踪

#### 2. MCP 服务 (services/mcp/)

**协议实现**:
- 多传输类型: stdio, SSE, HTTP, WebSocket, SDK-internal
- OAuth 2.0 + PKCE + XAA (Cross-App Access)
- 进程内传输用于同进程通信

**服务器管理**:
- 7 层配置作用域 (Enterprise > Local > Project > User > Dynamic > Plugin > Claude.ai)
- 自动重连 + 指数退避
- 过时客户端检测与清理

**工具集成**:
- 工具名空间化: `mcp__{server}__{tool}`
- 输出截断: 25k tokens
- 智能图片压缩
- 工具结果缓存

**资源处理**:
- `resources/list` 发现
- Channel notifications 推送消息到对话
- 权限请求流程

**安全特性**:
- 企业策略控制
- 项目级服务器审批
- OS keychain 安全 token 存储
- 多层认证

#### 3. 压缩服务 (services/compact/)

**4 种压缩策略**:
1. **Full Compaction** - 通过 forked agent 或流式 API 总结整个对话
2. **Partial Compaction** - 围绕 pivot 消息总结
3. **Session Memory Compaction** - 使用预提取的会话内存 (实验性)
4. **Micro-Compaction** - 清除旧工具结果不总结

**Token 管理**:
- Effective window = context_window - 20,000 (预留摘要输出)
- Auto-compact 触发 = effective_window - 13,000 buffer
- 阻塞限制 = effective_window - 3,000 (手动压缩)
- 3 次连续失败后熔断

**触发器**:
- Auto-compact: 每轮主动检查
- Time-based micro-compact: 30+ 分钟空闲后
- Cached micro-compact: GrowthBook 配置计数触发
- Manual: `/compact` 命令

#### 4. 分析服务 (services/analytics/)

**事件系统**:
- 队列式启动: 事件排队直到 sink 附加
- 类型安全元数据: 仅 `boolean | number | undefined`
- PII 保护: `_PROTO_*` 键剥离

**遥测管道**:
- 双 sink: Datadog 和 1P 事件日志
- OpenTelemetry-based
- 磁盘支持的重试队列
- 二次退避 (最多 8 次)

**GrowthBook 集成**:
- 服务端特性评估
- 多访问模式: 缓存、阻塞、安全门、授权检查
- 实验日志自动曝光跟踪
- Environment variable overrides

**隐私控制**:
- Analytics 禁用: test env, 3P 云提供商, 用户 opt-out
- Per-sink killswitch
- 工具名清理
